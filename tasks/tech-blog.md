# Task: 개인 기술 블로그

개인 기술 블로그를 기획하고 개발한다.

## 오케스트레이터

| 단계 | 오케스트레이터  | 설명               |
| ---- | --------------- | ------------------ |
| 기획 | `planning-team` | 기획 및 설계       |

## 프로젝트 방향

### 목표

- 개인 기술 블로그를 만든다.
- 블로그 주제/니치는 아직 미정 — 시장 조사를 통해 유망한 기술 블로그 주제를 탐색하고 추천받고 싶다.

### 기술 스택

- **프론트엔드**: React + Next.js
- **백엔드**: Node.js 또는 NestJS (NestJS는 처음 사용, 학습 곡선 고려 필요)
- **DB**: 후보로만 검토 (무료 티어 우선 — Supabase, PlanetScale, SQLite 등)

### 핵심 기능 (MVP)

- 게시글 작성 (마크다운 에디터)
- 게시글 공유 (SNS/링크)
- 카테고리 분류
- 검색
- 댓글 (MVP 이후 고려 가능)

### 디자인

- 방향 미정 — 레퍼런스 조사 및 추천 필요
- 깔끔하고 가독성 좋은 디자인 선호
- 모바일 반응형 필수

### 배포

- 빠르게 시작할 수 있는 환경 (Vercel 등 무료 호스팅 우선)
- 복잡한 인프라 설정 최소화

### 제약 조건

- **예산**: 거의 없음. 무료 티어 중심. 잘되면 점차 확대.
- **일정**: 틈틈이 작업. 데드라인 없음.
- **경험**: NestJS 미경험. 학습 비용을 고려한 기술 선택 필요.

### 시장 조사 요청 사항

- 개발자 기술 블로그 트렌드 (어떤 주제가 인기 있는지, 차별화 포인트)
- 유사 기술 블로그 플랫폼/서비스 비교 (velog, tistory, medium, dev.to, 자체 구축 등)
- 자체 구축 블로그의 장단점 및 현실적인 접근 전략
